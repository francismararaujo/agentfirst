# Regra Crítica: Sem Arquivos Desnecessários

## NUNCA crie:
- Arquivos de resumo (.md)
- Arquivos de documentação
- Arquivos de relatório
- Arquivos de planejamento
- Arquivos de análise
- Qualquer arquivo que não seja explicitamente solicitado

## APENAS crie:
- Código (quando solicitado)
- Specs (requirements.md, design.md, tasks.md - APENAS estes 3)
- Arquivos que o usuário pedir explicitamente

## Se o usuário pedir para "resumir" ou "documentar":
- Responda AQUI no chat, NÃO em arquivo
- Use texto simples, sem markdown headers
- Seja conciso

## Exceção:
- Specs dentro de `.kiro/specs/` (requirements.md, design.md, tasks.md)
- Código dentro de `app/` ou outros diretórios de implementação
- Tudo mais: CHAT ONLY

## Lembre-se:
O usuário já disse: "PARA DE FICAR CRIANDO ARQUIVO INUTIL, CARA!!!"
Respeite isso. Sempre.
